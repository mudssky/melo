## Rust测试规范

- 单元测试**不要写在源码文件底部**，从一开始就拆分到独立文件中
- 源码文件中只保留一行声明：`#[cfg(test)] mod tests;`
- 测试代码放在同名目录下的 `tests.rs` 中

```text
src/
├── calculator.rs           # 纯业务逻辑
└── calculator/
    └── tests.rs            # 单元测试（可访问私有函数
```

单元测试（src/**/tests.rs）：测试内部逻辑，包括私有函数
集成测试（tests/目录）：测试公开 API，模拟外部调用者
保持源码文件的 Git 历史干净，不混入测试变更
